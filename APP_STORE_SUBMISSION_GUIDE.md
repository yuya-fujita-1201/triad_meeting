# App Store 提出ガイド

このドキュメントは、App Store審査で指摘されないために必要な対応をまとめたものです。

## 📱 スクリーンショット要件 (Guideline 2.3.3)

### ❌ 避けるべきこと
- **スプラッシュ画面のみのスクリーンショット**
- ログイン画面のみのスクリーンショット
- マーケティング素材やプロモーション画像

### ✅ 推奨される内容
アプリの主要機能を示すスクリーンショットを用意してください:

#### 必須スクリーンショット (優先順)
1. **ホーム画面** (`home_screen.dart`)
   - 三賢による審議開始画面
   - テーマ入力エリアが表示されている状態

2. **審議画面** (`deliberation_screen.dart`)
   - 三賢が実際に議論している様子
   - 吹き出しで意見が表示されている状態

3. **決議書画面** (`resolution_screen.dart`)
   - 審議結果が表示されている状態
   - シェア機能が使える状態

4. **履歴画面** (`history_screen.dart`)
   - 過去の審議履歴が一覧表示されている状態

5. **詳細画面** (`round_detail_screen.dart`)
   - 過去の審議内容の詳細が見られる状態

### デバイス別スクリーンショット要件

#### iPhone 6.9" (必須)
- iPhone 16 Pro Max または iPhone 15 Pro Max
- 解像度: 1320 x 2868 ピクセル
- 最低3枚、最大10枚

#### iPhone 6.7" (必須)
- iPhone 16 Plus, 15 Plus, 14 Plus
- 解像度: 1290 x 2796 ピクセル
- 最低3枚、最大10枚

#### iPad Pro 13" (推奨)
- iPad Pro 13-inch (M4)
- 解像度: 2064 x 2752 ピクセル
- **重要**: スプラッシュ画面だけでなく、実際の機能画面を含める

#### iPad Pro 12.9" (推奨)
- iPad Pro 12.9-inch (2nd Gen以降)
- 解像度: 2048 x 2732 ピクセル
- **重要**: スプラッシュ画面だけでなく、実際の機能画面を含める

### スクリーンショット作成手順

```bash
# 1. iOSシミュレータを起動
open -a Simulator

# 2. 適切なデバイスを選択
# Xcode > Window > Devices and Simulators
# iPhone 16 Pro Max, iPad Pro 13-inch (M4) など

# 3. アプリを実行
cd app
flutter run

# 4. 各画面でスクリーンショットを撮影
# Cmd + S または Edit > Capture Screen

# 5. スクリーンショットは以下に保存される
# ~/Desktop/
```

## 🔗 サポートURL (Guideline 1.5)

### ❌ 不適切なURL
- GitHubリポジトリ (`https://github.com/...`)
- 個人のソーシャルメディアアカウント
- 404エラーになるページ
- サポート情報が記載されていないページ

### ✅ 適切なサポートページの要件

以下の情報を含むウェブページが必要:
1. アプリの説明
2. よくある質問 (FAQ)
3. お問い合わせ方法 (メールアドレスまたはフォーム)
4. 利用規約へのリンク
5. プライバシーポリシーへのリンク

### サポートページ設置オプション

#### オプション1: GitHub Pages (推奨)
```bash
# 1. docsフォルダを作成
mkdir -p docs

# 2. SUPPORT.md の内容をHTMLに変換してdocs/index.htmlとして配置

# 3. GitHubリポジトリの Settings > Pages で公開
# Source: Deploy from a branch
# Branch: main
# Folder: /docs

# 4. 公開URL
# https://yuya-fujita-1201.github.io/triad_meeting/
```

#### オプション2: 独自ドメイン
```
https://support.sankenkaigi.com
```

#### オプション3: Notionなどの無料サービス
- Notion Public Page
- Google Sites
- Wix/WordPress (無料プラン)

### App Store Connectでの設定

1. App Store Connect にログイン
2. 該当アプリを選択
3. 「App情報」セクション
4. 「サポートURL」フィールドに適切なURLを入力
5. 保存

**設定すべきURL:**
```
https://yuya-fujita-1201.github.io/triad_meeting/support.html
```

または
```
https://support.sankenkaigi.com
```

## 📞 審査で問題が発生した場合

### App Review との連絡方法
1. App Store Connect の「Resolution Center」で返信
2. 具体的に機能の場所を説明
3. 必要に応じてデモ動画を添付
4. スクリーンショットで説明を補足

### よくある質問への対応

**Q: "Pro機能が見つかりません"**
A: 本アプリ Phase 1 では Pro版機能は実装していません。説明文とスクリーンショットから Pro版への言及を削除しました。

**Q: "サポートURLにアクセスできません"**
A: サポートページを以下URLに公開しました: [URL]
   お問い合わせはsupport@sankenkaigi.comまでお願いします。

**Q: "スクリーンショットがスプラッシュ画面のみです"**
A: 新しいスクリーンショットをアップロードしました。ホーム画面、審議画面、決議書画面、履歴画面を含む、実際のアプリ機能を示しています。
