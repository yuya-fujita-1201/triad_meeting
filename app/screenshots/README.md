# App Store スクリーンショット

このディレクトリには、App Store審査用のスクリーンショットを保存します。

## 📱 ディレクトリ構成

```
screenshots/
├── iphone-6.9/      # iPhone 16 Pro Max / 15 Pro Max (1320 x 2868)
├── iphone-6.7/      # iPhone 16 Plus / 15 Plus (1290 x 2796)
└── ipad-13/         # iPad Pro 13-inch M4 (2064 x 2752)
```

## 📸 必要なスクリーンショット

各ディレクトリに以下のスクリーンショットを保存してください（最低3枚、最大10枚）:

### 推奨する順序

1. **1-home.png** - ホーム画面（テーマ入力エリアが表示されている状態）
2. **2-deliberation.png** - 審議画面（3人の賢者が議論している様子）
3. **3-resolution.png** - 決議書画面（審議結果が表示されている状態）
4. **4-history.png** - 履歴画面（過去の審議一覧）
5. **5-detail.png** - 詳細画面（過去の審議内容の詳細）

## 🎯 スクリーンショット要件

### ✅ 良いスクリーンショット
- アプリの主要機能を示している
- 実際の使用状態を示している
- UIが明確に見える
- 適切な解像度

### ❌ 避けるべきスクリーンショット
- スプラッシュ画面のみ
- ログイン画面のみ
- 空の画面
- エラー画面

## 🖼️ スクリーンショット撮影手順

### 1. シミュレータの起動

```bash
# Xcodeを起動してシミュレータを選択
# iPhone 16 Pro Max, iPad Pro 13-inch (M4) など

# または、コマンドラインから
open -a Simulator
```

### 2. アプリの実行

```bash
cd app
flutter run
```

### 3. スクリーンショット撮影

各画面で以下のいずれかの方法でスクリーンショットを撮影:

**方法1: キーボードショートカット**
```
Cmd + S
```

**方法2: シミュレータメニュー**
```
Edit > Capture Screen
```

スクリーンショットは `~/Desktop/` に保存されます。

### 4. ファイルの整理

撮影したスクリーンショットを適切なディレクトリに移動:

```bash
# デスクトップから移動
mv ~/Desktop/Screenshot*.png ./screenshots/iphone-6.9/1-home.png
```

## 📏 解像度の確認

スクリーンショットの解像度を確認:

```bash
# macOSの場合
sips -g pixelWidth -g pixelHeight screenshots/iphone-6.9/1-home.png
```

### 必要な解像度

| デバイス | 解像度 | 縦横比 |
|---------|--------|--------|
| iPhone 6.9" | 1320 x 2868 px | 9:19.5 |
| iPhone 6.7" | 1290 x 2796 px | 9:19.5 |
| iPad Pro 13" | 2064 x 2752 px | 3:4 |

## 🎨 スクリーンショット作成のコツ

### データの準備
実際の使用例を示すため、事前にサンプルデータを用意:

1. **審議テーマの例**:
   - "転職すべきか現職に残るべきか"
   - "結婚するタイミングについて"
   - "新しい趣味を始めるべきか"

2. **履歴データ**: 3-5件の過去の審議を作成

### タイミング
- 審議画面: 議論が進行中の状態で撮影
- 決議書画面: 完全な決議書が表示された状態
- 履歴画面: 複数のアイテムが表示されている状態

### 注意点
- 個人情報を含まない
- テスト用のダミーデータを使用
- 実際の機能を正確に反映

## 📤 App Store Connectへのアップロード

1. App Store Connect にログイン
2. 該当アプリの「App Store」タブを選択
3. 「メディアマネージャー」セクション
4. 各デバイスサイズのスクリーンショットをドラッグ&ドロップ
5. 順序を調整（最も重要な画面を最初に）

### アップロード前チェック
- [ ] 全デバイスサイズのスクリーンショットを用意
- [ ] 各デバイスで最低3枚以上
- [ ] 解像度が正しい
- [ ] ファイル形式がJPEGまたはPNG
- [ ] ファイルサイズが適切（各5MB以下推奨）

## 🔄 更新時の対応

アプリのUIを変更した場合:

1. 変更された画面のスクリーンショットを再撮影
2. 該当ファイルを上書き
3. App Store Connectで新しいスクリーンショットをアップロード
